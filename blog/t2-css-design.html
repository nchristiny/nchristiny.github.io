Week 2 Wrap up! HTML & CSS, class & id

Wherein I extend the water metaphor for far to long

June 21 2015

Between the devil and deep blue CSS

When it comes to CSS3 and HTML5, my previous knowledge was meager at best and this week I have truly learned "drinking from the firehouse". Drinking the sweet cascade... of Style Sheets!

One of the things we have learned as a best known method, standards compliant and just common sense action is to distinguish between the CSS selectors, class and id. I will make a humble attempt here to explain this now to the best of my abilities.

One of the ways CSS will determine via internal algorithm what style will "cascade" forth onto the browser page is called specificity. This property is determined by many factors, not the least of which is "Weight". Different selectors carry different weights, and id carries a specificity greater than that of class, thus gaining the upper hand if all else is equal when it comes time for the browser deciding what CSS rule to apply to a given attribute.

To illustrate this, since words are failing me at the moment, you can imagine that the CSS rules "cascade" along the path of least resistance, just like a waterfall (let's see just how many nautical miles we can wring from the water analogy). It helps to know that the general order of element weight is as follows from least to most ... "heavy".

Universal (*) < Tag < Class < Attribute < Pseudo < id < Inline style

Accordingly any CSS rule denoted by a simple HTML tag, say p for paragraph, will be overwritten by another rule that targets a paragraph with class because class is heavier and therefore more specific just so: p#class. Moving along the line, you may easily infer that as id is heavier than class, all things being equal it will always trump the class selector.

Another important difference is that the id attribute is supposed to be unique in an HTML document, while class can apply to many tags. In this way, the relative rarity of the id tag serves to remind the reader that it is more specific, the rara avis in the world of HTML attributes.

I hope this has been informative, and If you see any errors in this brief summary, please do not hesitate to send up a reply via the usual channels of communication and/or pillory me in the public square.

Thanks for reading. Have a great week!
