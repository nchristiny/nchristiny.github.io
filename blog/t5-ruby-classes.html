Ruby Classes

or: Everything's an object in Ruby. (Including classes.)

July 19 2015

Makin' babies with classes in Ruby

This week 5 at DBC, we get deep into Ruby classes. Some pretty heavy stuff which definitely has put me out of my comfort zone many times, but I'm rolling with the punches so far.

In the last couple of days the pieces have been falling into place, and I believe that I have been experiencing the beginnings of that dawning realization tI had seen described by seasoned Ruby vets.

So, in the spirit of Rubber duck debugging, I'd like to take a short jaunt into the wonderful world of Ruby classes.

Everything's an object in Ruby. Everything. Our objects in Ruby are able to do things - they walk and talk, live and breathe, by sending and receiving messages, or methods.

In order for them to understand the methods, the objects need to first know what they are, like a baby taking it's first steps - it's gotta learn how to crawl, and of what it's capable.

We teach our methods by intializing them in classes.

So let's put this into practice: We'll define a class of Object, Baby, intialize it, give it some methods within the class defintion. Afetrwards, we'll call it with those methods and see what else we can do.

class Baby   #Our brand new class of object
          def initialize(age, name)  #we give the new class some instance variables
            @age = age
            @name = name
          end

          def cry  #define some methods! 
            puts "waaahh!"
          end

          def walk
            if @age > 1
              puts "Yay! Looks who's walking"
            else
              puts "Keep crawling kid"
            end
          end

          def who
            puts "GOO GOO GA!"
            puts "Translation: I am, my name is #{@name},"
            puts "and I am #{@age},"
            puts "and I'm going to be the next Amelia Earhart."
          end

        end

We then define our new object, a new baby!

baby = Baby.new(1.5,"Athena")

(Implicit return)=> #

And finally we can see how it reacts to some of the class methods we jsut created.

baby.cry

Output: waaahh! 

baby.walk

Output: Yay! Looks who's walking

And finally:

baby.who

Output: GOO GOO GA! Translation: I am, my name is Athena, and I am 1.5, and I'm going to be the next Amelia Earhart.

In addition to the Ruby class methods you have defined, you can use .methods to list the built-in methods that your newly created class is capable of accepting.

As you can see, defining a class let's you create pretty much any object you want! Pretty neat.
