I'm feeling up to the task of attempting to explain the main difference between an array and hashes in Ruby.

From what I have gathered, in my extremely limited time as a student of the subject, an array consists of an ordered group of objects, (indexed by sequential integers,) as opposed to a hash which can be assigned any old index imaginable, numbers, strings, different types of fruit. The main thing is that hashes are not ordered, and arrays are. Edit: It is not strictly true that hash objects are unordered in recent versions of Ruby 1.9 onwards; hash preserves the insertion order, that is the order in which the collection was created originally. I am sure that there is a great reason for this, but I am very new to the art of Ruby programming, but I don't really understand it yet unfortunately. At first glance it would seem to violate the principle of least astonishment. This is due to being able to traverse the hash more efficiently, or something like that. We work with what we have, so I hereby amend my previous statement: both hash and array objects in Ruby preserve order.

Apart from that they seem to able to be worked over with the usual Ruby heavy duty tools, methods and their arguments.

Right now we shall be enjoying this glorious 4th of July weekend without work and be back, with a vengeance.

Arrays are all like: 

array = [1, "two", 3.0] #=> [1, "two", 3.0]

#As you can see, well ordered to the point of fastidiousness

but Hashes are all:

Hashes = {"Bro" => 21, "Random" => 929 }

#Hashes really do not care, do they?
